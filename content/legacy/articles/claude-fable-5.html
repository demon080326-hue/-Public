Claude Fable 5 完整深度彙整

免費使用倒數、模型定位、效能差距、定價、下架事件、安全分類器與使用建議一次整理。

▋ 最重要：免費使用倒數計時

付費方案（Pro / Max / Team / Enterprise）現在可以免費使用 Fable 5，截止時間是 2026 年 7 月 13 日 14:59，距今只剩約 5 天。上限是每週用量額度的 50%，超過後可切換 Opus 4.8 或 Sonnet 5 繼續，或買 Usage Credits 繼續用 Fable 5。免費方案完全無法使用 Fable 5。

▋ Fable 5 是什麼？

Claude Fable 5 是 Anthropic 在 2026 年 6 月 9 日推出的最新旗艦模型，是目前對外公開的最強 Claude 模型，定位在 Opus 4.8 之上，屬於「Mythos 級」。同場推出還有一個叫 Claude Mythos 5 的版本，兩者使用相同的底層模型權重，差別在於 Mythos 5 沒有安全分類器限制，但僅開放給通過 Anthropic「Project Glasswing」審核的特定政府與研究機構，一般用戶無法使用。

基本規格：

  • 支援 100 萬 token 上下文視窗
  • 單次最多輸出 128,000 tokens
  • 支援平台：claude.ai 網頁版、桌面版、手機 App、Claude Code、Cowork、AWS Bedrock、Google Cloud、Microsoft Foundry
  • API 代號：claude-fable-5

▋ 比 Opus 4.8 強多少？

評測項目 Fable 5 Opus 4.8 差距
SWE-Bench Pro（程式修 bug） 80.3% 69.2% +11.1%
FrontierCode（複雜程式） 29.3% 13.4% +15.9%
BenchLM 綜合評分 92 86 +6

重點：任務越長、越複雜，Fable 5 的優勢越明顯。短任務或日常對話，兩者差距縮小很多。

實際用戶案例：

  • Replit 表示 Fable 5 在「vibe-coding」基準測試中表現最高
  • 某金融客戶的核心分析基準從 Opus 4.8 跳升 10 個百分點，首次突破 90%
  • 一家試算表自動化公司發現 Fable 5 不只更準確，還快了 25–30%

▋ 定價比較

項目 Fable 5 Opus 4.8
輸入（每百萬 token） $10 美元 $5 美元
輸出（每百萬 token） $50 美元 $25 美元
5 分鐘快取寫入 $12.50 $6.25
1 小時快取寫入 $20 $10

整體約是 Opus 4.8 的兩倍。不過 Anthropic 和客戶都反映，Fable 5 完成任務所需的 token 數更少、對話輪次更少，實際費用差距可能比帳面小。

▋ 事件始末：被下架 19 天的前因後果

6/9 上線：Fable 5 與 Mythos 5 正式推出，引發業界高度關注。

6/12 全球下架：上線僅三天，Amazon 研究人員發現了一個嚴重的越獄手法：透過提示 Fable 5 閱讀程式碼庫並找出軟體漏洞，在某些情況下模型會直接產出示範如何利用漏洞的攻擊程式碼。美國商務部工業和安全局（Bureau of Industry and Security）隨即對兩款模型發出出口管制命令，要求 Anthropic 在將模型「出口」給任何外國人之前必須取得政府許可，包括 Anthropic 自己的非美籍員工。Anthropic 被迫暫停全球所有人的存取，這在 AI 業界是前所未有的事。

6/30 解禁：美國商務部解除出口管制。Anthropic 同步宣布已訓練出針對該越獄手法的新版安全分類器，並與政府共同提出「越獄嚴重性分級框架」。

7/1 全球恢復：Fable 5 在網頁版、桌面版、手機 App、Claude Code、Cowork 及 API 全面恢復，同步開放補償優惠。

▋ 恢復後的 Fable 5 有什麼不同？

新版加入了三個方向的安全分類器：

  1. 網路安全（cybersecurity）：攔截可用於攻擊的安全漏洞請求
  2. 生物/化學（bio/chem）：攔截潛在危害性的生物化學內容
  3. 蒸餾保護（distillation）：防止用來竊取模型能力的提示

官方說明：攔截越獄手法的準確率超過 99%。當請求觸發分類器時，Fable 5 會將請求交給 Opus 4.8 回答，使用者會收到通知，且不會以 Fable 5 的費率計費。根據 Anthropic 數據，這種情況在不到 5% 的會話中發生。

一般日常用途（聊天、寫作、寫程式、分析資料）幾乎完全不受影響。

開發者注意：Fable 5 被分類器拒絕時，API 回傳的是 HTTP 200 搭配 stop_reason: "refusal"，而非錯誤代碼，建議在程式中加入 fallback 機制。

▋ 各優惠完整時程

優惠內容 適用對象 截止時間
每週額度 50% 免費用 Fable 5 Pro / Max / Team / Enterprise 7/13 14:59（剩 5 天）
Claude Code 每週上限提升 50% Pro / Max / Team / Enterprise 7/13
Cowork 五小時用量翻倍 Pro / Max / Team 已截止（7/5）

▋ 7/13 之後怎麼辦？

付費方案對 Fable 5 的存取改為 Usage Credits 計費，用多少扣多少；API 定價維持輸入 $10、輸出 $50 每百萬 token。不常用的話，切回 Opus 4.8 或 Sonnet 5 最省錢。

▋ 免費用戶的替代方案：Sonnet 5

Anthropic 在 6/30 同步推出 Claude Sonnet 5，已成為免費用戶的預設模型。效能逼近 Opus 4.8，但成本更低（每百萬 token 輸入約 $3、輸出 $15），日常寫作、問答、簡單分析全部夠用。

▋ 誰應該用 Fable 5？誰不需要？

值得用 Fable 5 的情境：

  • 長文件分析、大型程式碼重構、多步驟 agent 自動化任務
  • 需要深度推理的研究型任務
  • 之前用 Opus 4.8 卡住或品質不滿意的困難任務

用 Opus 4.8 或 Sonnet 5 就夠的情境：

  • 日常聊天、摘要、翻譯、改寫
  • 高頻次、大量的 API 呼叫（成本考量）
  • 對回應速度有要求的場景
